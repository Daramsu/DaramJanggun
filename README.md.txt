local 지역적 자기컴퓨터에 있는 레퍼지토리
remote에 올려

remote에 여러명의 프로그래머가 접근할 수 있어

여러 프로그래머가 remote에 있는코드를 함께 수정하면서
'레퍼지토리' 째로 수정을하는거야

remote 레퍼지토리를 어디에 저장하느냐?

같이 일하는사람끼리 합의된 서버컴퓨터일수도있고

         원격 레퍼지토리에서 바뀐 상태를 (다른사람이 코드 바꾼거를)내컴에다가 반영할수있어

깃허브 사이트( 인터넷에있는 고양이 사이트)
깃허브 데스크탑( 컴퓨터에있는 고양이 프로그램)

깃허브데스크탑 핵기능 3가지!
오른쪽 위 Fetch origin = 깃 허브 사이트에서 화둥이 컴푸터쪽으로 받아오기
왼쪽 아래 commit = 화둥이가 변경한 내용을 화둥이 컴퓨터의 깃에 저장!
오른쪽 위 push = 화둥이가 커밋한걸 깃허브 사이트로 올려보내기
